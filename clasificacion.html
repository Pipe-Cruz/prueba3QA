Clasificación de QA

Dentro del campo de aseguramiento de calidad del software, se reconocen dos roles, el QA funcional y el QA automatizador, los cuales tienen responsabilidades y enfoques ligeramente diferentes en el proceso de pruebas de software. A continuación se presentan algunas diferencias y herramientas que suelen utilizar:

QA Funcional

  • Enfoque:

    El QA funcional se centra en la evaluación de la funcionalidad del software para asegurarse de que cumpla con los requisitos y las expectativas del cliente. Esto implica realizar pruebas manuales para identificar errores y verificar que el software funcione según lo previsto.

  • Responsabilidades:

    El QA funcional crea casos de prueba, ejecuta pruebas manuales, registra problemas y colabora con el equipo de desarrollo para resolverlos. También participa en la revisión de requisitos y documentación.

  • Herramientas:

    Los QA funcionales utilizan herramientas de seguimiento de problemas, herramientas de gestión de pruebas y herramientas de captura de pantallas y grabación de pruebas.

    • Herramientas para el seguimiento de problemas: JIRA
    • Herramientas para la gestión de pruebas: TestRail
    • Herramientas para captura de pantalla y grabación de pruebas: Snagit - Camtasia.

QA automatizador

  • Enfoque:

    El QA automatizador se enfoca en automatizar pruebas mediante el desarrollo de scripts o casos de prueba automatizados. Su objetivo es aumentar la eficiencia de las pruebas, especialmente en áreas repetitivas o de regresión.

  • Responsabilidades:

    El QA automatizador crea scripts de prueba automatizados utilizando lenguajes de programación o herramientas específicas de automatización. Ejecuta pruebas automatizadas y mantiene los scripts a medida que el software evoluciona.

  • Herramientas:

    Los QA automatizadores utilizan herramientas de automatización de pruebas, herramientas de automatización de carga y lenguajes de programación para desarrollar scripts de prueba; esto dependiendo del tipo de prueba a automatizar.

    • Herramientas para la automatización de pruebas: Selenium - Appium - TestComplete
    • Herramientas para la automatización de carga: JMeter
    • Lenguajes de programación: Java - Python - C#
